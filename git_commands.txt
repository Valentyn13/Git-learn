git init

git add .

git commit -m ""

git push - отправить в гитхаб

git branch - получаем ветку

git branch name - сщздаем ветку

git branch -d branch_name - удалить ветку

git push origin branch_name - пушим ветку в гитхаб

git push origin --delete branch_name - удаляем ветку с гитхаба

git merge branch_name - слияние веток
(
например мы находимся в master и пишем эту команду
тогда ветка branch_name вольется в master
!!!! ПОСЛЕ СЛИЯНИЯ ВЕТКУ branch_name УДАЛЯЮТ
)

Merge против rebase

git merge branch_name - переливание коммитов, создается новый коммит (merge commit) в котором все подтянутые изменения

git rebase branch_name - вытягиваются коммиты из мастера и в конец добавляются все коммиты из твоей ветки
(
    Важно
    Если используешм rebase то для пуша на удаленную ветку теперь надо будет использовать
    git push origin branch_name --force
    могут быть проблемы если с веткой работают несколько человек
    такой подход шорош если ты в одиночку работаешь с веткой
)
